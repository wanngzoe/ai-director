## 待优化方向

### 🔴 高优先级：工作台层级关系优化

**问题描述**：
- 当前工作台是所有项目的入口，但"剧本-选角-场景-道具-分镜&视频生成-后期-成片"都属于对某个特定项目的操作
- 层级关系不清晰，用户在不同项目间切换时容易混淆

**优化目标**：
1. 层级关系更清晰
2. 不同项目间更好切换，且互不干扰

**可能的解决方案**：

| 方案 | 描述 | 优缺点 |
|------|------|--------|
| **方案A：项目标签栏** | 顶部添加项目切换标签栏，类似于浏览器标签页，不同项目用标签页切换 | 直观但占用顶部空间 |
| **方案B：左侧项目列表** | 左侧sidebar分为两层：顶层项目列表，点击后展开该项目的子菜单（剧本/选角等） | 符合导航习惯 |
| **方案C：面包屑导航** | 每个页面顶部显示面包屑：项目名 > 当前模块 | 轻量但不够直观 |
| **方案D：全局项目选择器** | 右上角添加项目下拉选择器，切换后整个工作区内容变化 | 简单但切换成本高 |
| **方案E：工作区模式** (选定) | 点击项目进入"工作区"，左侧显示项目操作菜单；点击项目卡片返回项目列表 | 交互更沉浸，层级清晰 |

### 方案E详细设计

**两种视图模式**：
1. **项目列表视图**（当前工作台）
   - 显示所有项目卡片
   - 点击项目进入工作区视图

2. **工作区视图**（项目内操作）
   - 左侧sidebar显示当前项目的操作菜单（剧本/选角/场景/道具/分镜/后期/成片）
   - 顶部显示当前项目名称 + 返回项目列表按钮
   - 右上角可切换其他项目（进入其他项目的工作区）

**预期交互流程**：
```
工作台（项目列表）
    ↓ 点击项目A
项目A的工作区
    ├── 剧本 → 剧本工坊
    ├── 选角 → 选角中心
    ├── 场景 → 场景工坊
    ├── 道具 → 道具管理
    ├── 分镜 → 分镜导演台
    ├── 后期 → 后期精修
    └── 成片 → 成片输出
    ↓ 点击返回/切换项目
项目B的工作区 / 项目列表
```
